# HR Data 원큐 Agent 핸즈온 가이드

> SK케미칼 | GPT-5 Chat | 비전공자 대상 | 약 120분

---

## 개요

사번을 입력하면 HR 인사·평가 데이터를 자동 조회하고, AI가 역량모델 기준에 따라 분석한 뒤, 사람이 검토·승인하면 Word 문서로 결과를 생성하는 에이전트를 만듭니다.

**전체 흐름:**
사번 입력 → 인사정보 API → 평가 API → AI 역량 분석 → HITL 승인 → Word 문서 생성 → 완료

**핸즈온 구성 (프로덕션 대비 단순화):**

| 항목 | 프로덕션 | 핸즈온 |
|------|----------|--------|
| 데이터 저장 | Dataverse + 동기화 | API 직접 호출 (저장소 없음) |
| 평가기준 | Dataverse 테이블 | Excel → 참조 자료(Knowledge) |
| HR API | 실제 HR 시스템 | Mock API 서버 |
| 문서 생성 | SharePoint + 전체 필드 | OneDrive + 핵심 필드만 |
| AI 모델 | GPT-5 Chat | GPT-5 Chat (동일) |

**Mock API:**
- Base URL: `https://mockhrserver.onrender.com`
- 인사정보 조회: `GET /api/v1/inhr/employees/{id}`
- 평가/리더십 조회: `GET /api/v1/myhr/evaluations/{id}`
- Swagger 문서: `https://mockhrserver.onrender.com/docs`
- 직원 100명, 23개 부서

**테스트용 사번:**

| 사번 | 이름 | 소속 | 직급 |
|------|------|------|------|
| EMP001 | 김민수 | 경영기획팀 | 부장 |
| EMP002 | 이서연 | 인사팀 | 차장 |
| EMP003 | 박준혁 | 연구개발팀 | 과장 |

## STEP 4. 사용자 지정 커넥터 생성 (HR API 연결)

> **커넥터(Connector)** = 에이전트가 외부 API와 통신하는 연결 통로. Swagger 문서를 가져오면 API 기능이 자동 등록됩니다.

### 커넥터 만들기

1. https://make.powerautomate.com 접속
2. 왼쪽 메뉴 **"사용자 지정 커넥터"**
3. **"+ 새 사용자 지정 커넥터"** → **"URL에서 OpenAPI 가져오기"**
4. 입력:

| 항목          | 입력 값                                             |
| ----------- | ------------------------------------------------ |
| 커넥터 이름      | `HR API`                                         |
| Swagger URL | `https://mockhrserver.onrender.com/swagger.json` |

> ⚠️ 반드시 `/swagger.json` (v2.0) 사용. `/openapi.json`은 v3.1.0이라 오류 발생. Power Automate는 Swagger 2.0만 지원.

5. **"가져오기"** → **"계속"**
6. 호스트가 `mockhrserver.onrender.com` 인지 확인 → **"보안 →"**
7. API로 구현되는 인증 선택: "인증 없음" 확인 → **"정의 →"** 
8. 좌측 "동작" 항목 중 "특정 임직원 전체인적정보 조회", "특정 임직원 평가/리더십 데이터 조회" 만 사용합니다.
9. "커넥터 만들기" 클릭
10. **"5. 테스트"** 탭 → **"+ 새 연결"** 클릭하여 연결 생성

### 🧪 커넥터 테스트

2. `employee_id`에 `EMP070` 입력
3. **"테스트 작업"** 클릭
4. **응답 200 + JSON 데이터 표시 = 성공**
5. **"14. get_employee_basic_by_id_api_v1_inhr_employees_basic__employee_id__get"** 선택
6. `employee_id`에 `EMP070` 입력
7. **"테스트 작업"** 클릭
8. **응답 200 + JSON 데이터 표시 = 성공**

## STEP 5. 커넥터를 도구(Tool)로 등록

3. "HR API" 클릭 → 아래 **2개 작업만** 선택:
   - ✅ **특정 임직원 전체인적정보 조회** (inHR — 기본정보, 가족, 해외, 자격증)
   - ✅ **특정 임직원 평가/리더십 데이터 조회** (myHR — 3개년 평가, 리더십 서베이)

## STEP 6. 토픽 구성 — 사번 입력

> **토픽(Topic)** = 사용자의 요청을 처리하는 대화 흐름. 순서도를 만든다고 생각하세요.

### 토픽 생성 및 트리거

1. 상단 **"토픽"** 탭 → **"+ 토픽 추가"**  → **"새로 시작"**
2. 토픽 이름: `프로필 생성`
3. 맨 위 **트리거 노드** 클릭 → **"토픽이 수행하는 작업 설명"** 입력:

```
사번을 입력받아 HR 데이터를 조회하고, 역량모델 기준에 따라 구성원 프로필을 분석합니다.
```

> 💡 이전 버전에는 "트리거 문구 추가" 버튼이 따로 있었지만, 현재는 자연어 설명 방식. AI가 "프로필 만들어줘", "역량 분석해줘" 등에 자동 매칭.

### 사번 입력받기

4. 트리거 아래 **"+"** → **"질문하기"**
5. 질문 텍스트:

```
분석할 구성원의 사번을 입력해주세요. (예: EMP001)
```

6. **식별** → **"사용자의 전체 응답"** 선택
7. **"다음 이름으로 사용자 응답 저장"** → 변수명을 `사번`으로 변경

### 시스템 토픽 연결 (대화 시작)

대화가 시작되면 자동으로 프로필 생성 토픽이 실행되도록 연결합니다. 이렇게 하면 이후 중간 테스트할 때 편리합니다.

9. 상단 **"토픽"** 탭 → **"시스템"** 탭 → **"대화 시작"** 토픽 열기
10. 기존 메시지 노드 아래 **"+"** → **"토픽 관리"** → **"다른 토픽으로 이동"** → **"프로필 생성"** 선택
11. **"저장"**

## STEP 7. 토픽 구성 — API 호출

### 인사정보 API 호출 (도구 노드 1)

1. 질문 노드 아래 **"+"** → **"도구 호출"**
2. **"특정 임직원 전체인적정보 조회"** 선택
3. 입력의 `employee_id` → `{x}` 클릭 → **"사용자 지정"** 탭 → `사번` 변수 선택
### 인사정보 결과 저장 (변수 값 설정)

두 API 모두 출력 변수명이 `data`로 동일하기 때문에, 두 번째 API 호출 시 첫 번째 결과가 덮어씌워집니다. 이를 방지하기 위해 먼저 별도 변수에 저장합니다.

4. 인사정보 도구 노드 아래 **"+"** → **"변수 관리"** → **"변수 값 설정"**
5. **"설정할 변수"** → **"새 변수 만들기"** → 변수명: `인사정보` (문자열 타입)
6. **"받는 사람 값"** 오른쪽 **"..."** → **"수식"** 탭 → 아래 수식 입력 후 **"삽입"**:

```
JSON(Topic.data)
```

### 평가정보 결과 저장 (변수 값 설정)

10. 평가/리더십 도구 노드 아래 **"+"** → **"변수 관리"** → **"변수 값 설정"**
11. **"설정할 변수"** → **"새 변수 만들기"** → 변수명: `평가정보` (문자열 타입)
12. **"받는 사람 값"** 오른쪽 **"..."** → **"수식"** 탭 → 아래 수식 입력 후 **"삽입"**:
```
JSON(Topic.data)
```

### 사번 검증 (조건 분기)

잘못된 사번을 입력하거나 존재하지 않는 직원인 경우, 안내 메시지를 보여주고 사번 입력으로 돌아가도록 합니다.

7. 변수 값 설정 노드 아래 **"+"** → **"조건 추가"** → **"조건"**
8. 조건 설정:
    - 왼쪽: 변수 선택 → `Topic.인사정보`
    - 연산자: **"같음"**
    - 오른쪽: **"..."** → **"수식"** 탭 → `Blank()` 입력 후 삽입
9. **조건에 맞는 경우 (왼쪽 분기)** — 사번이 잘못된 경우:
    - **"+"** → **"메시지 보내기"** → 텍스트: `해당 사번의 직원을 찾을 수 없습니다. 사번을 다시 확인해주세요.`
    - 메시지 아래 **"+"** → **"토픽 관리"** → **"토픽으로 리디렉션"** → **"프로필 생성"** 선택 (처음으로 돌아감)
10. **조건에 맞지 않는 경우 (오른쪽 분기)** — 정상 데이터인 경우: 이 분기 아래에 이후 모든 단계(평가 API 호출 ~ 끝)를 넣습니다.

11. 위치: **"OneDrive for Business"**, 문서 라이브러리: **"OneDrive"**
12. 파일: `/HR_프로필/케미칼용_결과양식_template.docx`
13. 고급 매개 변수에서 6개 핵심 필드 매핑:

| 필드 | 값 |
|------|-----|
| 성명_생년월 | JSON 구문 분석 → `name` |
| 소속 | JSON 구문 분석 → `department` |
| 직책_연차 | JSON 구문 분석 → `position` |
| 주요역량_직무전문성 | 변수 → `주요역량` |
| 상위역할_준비도 | 변수 → `상위역할` |
| 종합의견 | 변수 → `종합의견` |

> 💡 노란색 경고가 표시될 수 있으나 실행에는 영향 없습니다.

### 파일 만들기 (OneDrive에 저장)

14. **"+ 새 단계"** → **"비즈니스용 OneDrive"** → **"파일 만들기"**
15. 설정:

| 항목     | 값                                                                                                               |
| ------ | --------------------------------------------------------------------------------------------------------------- |
| 폴더 경로  | `/HR_프로필`                                                                                                       |
| 파일 이름  | 수식 - concat('hrprofile_', body('JSON_구문_분석')?['name'], '_', formatDateTime(utcNow(),'yyyyMMddHHmmss'), '.docx') |
| 파일 콘텐츠 | Word 템플릿의 **"Microsoft Word 문서"** 출력                                                                            |

### 공유 링크 만들기

생성된 파일의 공유 링크를 만들어 에이전트 사용자가 바로 열 수 있도록 합니다.

14. **"+ 새 단계"** → **"비즈니스용 OneDrive"** → **"공유 링크 만들기"** 선택
15. 설정:

| 항목    | 값                             |
| ----- | ----------------------------- |
| 파일 ID | 파일 만들기 → **"ID"** (동적 콘텐츠)    |
| 링크 유형 | **"View"**                    |
| 링크 범위 | **"조직"** (같은 회사 사용자가 볼 수 있도록) |

### 에이전트에게 응답

16. 마지막 **"에이전트에게 응답"** 단계에 텍스트 출력 추가:

| 항목 | 값 |
|------|-----|
| 출력 이름 | `파일링크` |
| 타입 | **텍스트** (반드시 텍스트로 설정) |
| 값 | 공유 링크 만들기 → **"웹 URL"** |

> ⚠️ 출력 타입을 반드시 **"텍스트"**로 설정하세요. "파일" 타입이면 Copilot Studio에서 FlowActionBadRequest (타입 불일치) 오류가 발생합니다.

17. **"저장"** 클릭

## STEP 12. 흐름 연결 및 최종 테스트

### 프로필 생성 토픽에 흐름 연결

1. **"프로필 생성"** 토픽 열기
2. 승인 경로의 메시지 노드 위에 **"+"** → **"도구 추가"** → **"HR 프로필 Word 생성"** 선택
3. 입력 3개 매핑:

| 흐름 입력        | 매핑 값 (수식)            |
| ------------ | -------------------- |
| 인사정보 (텍스트)   | 변수 선택 → `Topic.인사정보` |
| 평가정보 (텍스트 1) | 변수 선택 → `Topic.평가정보` |
| 분석결과 (텍스트 2) | 변수 선택 → `Topic.응답결과` |

4. 흐름 노드 아래 메시지에 파일링크 변수 포함:

```
프로필 문서가 생성되었습니다. 아래 링크에서 확인하세요:
{x} 파일링크
```

> 💡 파일링크 변수에는 OneDrive 공유 링크(URL)가 들어있어, 사용자가 클릭하면 바로 Word 문서를 열 수 있습니다.

5. **"저장"**

### 완성된 토픽 흐름

```
🎯 트리거          ← "프로필 생성해줘" 등 사용자 발화 매칭
   ↓
❓ 사번 질문        ← 사용자 응답 → Topic.사번 변수에 저장
   ↓
🔌 inHR API 호출   ← 기본정보, 가족, 해외경험, 자격증
   ↓
🔌 myHR API 호출   ← 3개년 평가등급, 리더십 서베이
   ↓
🤖 생성형 답변      ← 참조 자료 + API 데이터 → 역량 프로필 작성 → 응답결과 변수 저장
   ↓
👤 HITL 승인 요청   ← 승인 / 수정 요청 선택
   ↓
📄 Power Automate  ← (승인 시) Word 템플릿 채우기 → OneDrive 저장 → 공유 링크 생성
   ↓
✅ 완료             ← 생성된 Word 파일 링크 전달
```

### 🧪 최종 테스트

1. 테스트 채팅 → 🔄 다시 시작
2. `프로필 생성해줘` 입력
3. 사번 `EMP001` 입력
4. AI 역량 분석 결과 표시 확인
5. "승인" 선택
6. Power Automate 흐름 실행 → **OneDrive 공유 링크** 표시 확인
7. 링크 클릭하여 Word 문서가 정상적으로 열리는지 확인
8. OneDrive `/HR_프로필/` 폴더에 결과 파일 생성 확인

**✅ Word 문서가 생성되고 링크가 표시되면 핸즈온 완료!**

## 참고: Power Fx 수식

| 하고 싶은 것 | 올바른 수식 |
|-------------|------------|
| 문자열 합치기 | `"A" & "B"` (`Concat()` 아님) |
| Record → 문자열 | `JSON(변수)` (`Text()` 아님) |

## 트러블슈팅

| 증상 | 해결 방법 |
|------|----------|
| 커넥터가 도구 목록에 안 나옴 | 1~2분 대기 후 새로고침 |
| "할당된 변수: Record" 오류 | `JSON()`으로 감싸기 |
| Power Automate 흐름이 도구에 안 나옴 | 에이전트와 같은 솔루션에 추가 + 계획을 "Copilot Studio"로 변경 |
| Word 템플릿 필드 인식 실패 | 콘텐츠 컨트롤(SDT) 설정 확인 |
| FlowActionBadRequest (타입 불일치) | 흐름 출력 타입과 Copilot Studio 기대 타입 일치시키기 |
| Mock API 응답 없음 | 서버 슬립 — 30초 후 재시도 |
